\## 0. Codex向け最重要方針



このプロジェクトは、Godot 4.6で制作するグリッド制パズルゲームである。



最初の目標は、\*\*外壁ステージだけで遊べる v0.1 プロトタイプ\*\*を作ること。



v0.1では、以下に集中する。



\* 外壁グリッドを表示する

\* プレイヤーを1マスずつ移動する

\* ハシゴを左右にスライドする

\* プレイヤーがハシゴを登り降りする

\* 汚れた窓を掃除する

\* 猫が窓から見ている

\* Undo / Resetを実装する

\* JSONからステージを読み込む

\* 日本語・英語ローカライズに備える



v0.1では、以下は実装しない。



\* 室内ステージ

\* 窓から中に入る処理

\* 伸縮ハシゴ

\* 開閉窓

\* 箱押し

\* 猫の移動AI

\* Phaser製ステージエディタ

\* agent-sprite-forgeによる本番素材生成

\* 複雑なアニメーション

\* 物理演算



まずは「ハシゴを動かして窓を掃除するだけで楽しいか」を確認する。

日本語タイトル案:



```text

```



英語タイトル案:



```text

```



\### 1.2 ジャンル



```text

2Dグリッド制パズル

```



\### 1.3 コアコンセプト



プレイヤーは塔の窓掃除係。



スライド式ハシゴを左右に動かし、塔の外壁を登り、汚れた窓を掃除する。



将来的には、窓から塔の中に入り、内部パズルを解いて別の窓から外に出る。



v0.1では外壁のみを実装する。



\### 1.4 プレイヤー体験の目標



目指す体験は以下。



```text

見れば何をすればいいか分かる。

少し動かすと詰まりどころが分かる。

手を戻して試すうちに、ひらめいて解ける。

```



説明文を読ませるゲームではなく、画面内の配置で直感的に理解できるパズルにする。

\## 4. ゲームルール v0.1



\### 4.1 盤面



v0.1では外壁のみ。



盤面は2Dグリッド。



座標系は以下。



```text

x: 左から右へ増える

y: 上から下へ増える

```



例:



```text

(0,0) (1,0) (2,0)

(0,1) (1,1) (2,1)

(0,2) (1,2) (2,2)

```



\### 4.2 セル種別



v0.1で扱うセル種別。



| 種別           | 説明                   |

| ------------ | -------------------- |

| empty        | 何もない外壁。基本的には立てない     |

| ground       | 地面。プレイヤーが立てる         |

| ledge        | 足場。プレイヤーが立てる         |

| wall         | 壁。背景扱い               |

| window\_dirty | 汚れた窓。掃除対象            |

| window\_clean | 掃除済み窓                |

| cat\_window   | 猫が見ている窓。v0.1では演出兼ヒント |



\### 4.3 プレイヤー



プレイヤーは以下に立てる。



\* ground

\* ledge

\* ladder上のセル



プレイヤーは以下には立てない。



\* empty

\* wall

\* window\_dirty

\* window\_clean

\* cat\_window



ただし、窓の隣またはハシゴ上の同じ高さから掃除できる。



\### 4.4 ハシゴ



v0.1のハシゴは固定長。



\* 垂直方向のオブジェクト

\* 長さはステージごとに指定

\* 横方向に1マスずつスライドできる

\* ハシゴが占有するセルは、プレイヤーが上下移動できる

\* ハシゴの一番下はgroundまたはledgeに接している必要がある



v0.1では、ハシゴは伸縮しない。



\### 4.5 ハシゴの移動



プレイヤーがハシゴの下端付近にいるとき、左右に押せる。



実装を単純にするため、v0.1では以下の仕様にする。



\* プレイヤーが地面または足場にいる

\* プレイヤーがハシゴの下端セルの左右どちらかに隣接している

\* 操作入力でハシゴを左右に1マス移動できる

\* 移動先のハシゴ占有セルが盤面内である

\* 移動先に障害物がない

\* 移動後もハシゴ下端がgroundまたはledgeに接している



ハシゴ上にプレイヤーがいる場合、v0.1ではハシゴを動かせない。



将来的には「ハシゴに乗ったままスライド」を検討するが、v0.1では実装しない。



\### 4.6 掃除



汚れ窓は、プレイヤーが掃除可能位置にいるときに掃除できる。



掃除可能条件:



\* プレイヤーと汚れ窓が上下左右に隣接している

\* または、プレイヤーがハシゴ上にいて、同じ高さの左右隣に汚れ窓がある



掃除操作を実行すると、対象の窓は `window\_dirty` から `window\_clean` になる。



すべての `window\_dirty` がなくなったらクリア。



\### 4.7 猫



v0.1の猫は動かない。



猫は `cat\_window` として配置する。



役割:



\* 見た目の魅力

\* プレイヤーへのゆるいヒント

\* 将来的なギミックの予告



v0.1では猫に当たり判定やAIは不要。

\## 5. 操作仕様



\### 5.1 キーボード



| 入力            | 動作              |

| ------------- | --------------- |

| ← / A         | 左へ移動            |

| → / D         | 右へ移動            |

| ↑ / W         | 上へ移動、またはハシゴを登る  |

| ↓ / S         | 下へ移動、またはハシゴを降りる |

| Space / Enter | 決定、掃除           |

| Shift + ←     | ハシゴを左へ押す        |

| Shift + →     | ハシゴを右へ押す        |

| Z             | Undo            |

| R             | Reset           |

| Esc           | ポーズ、またはメニュー     |



\### 5.2 v0.1の簡易操作



最初の実装では、以下でもよい。



\* 方向キーでプレイヤー移動

\* ハシゴの横にいる状態でShift + 左右でハシゴ移動

\* Spaceで掃除

\* ZでUndo

\* RでReset



スマホ操作はv0.1では不要。

\## 7. Undo / Reset仕様



\### 7.1 Undo



Undoは1手戻し。



1手とは以下のいずれか。



\* プレイヤー移動

\* ハシゴ移動

\* 掃除

\* ステージ状態を変える操作



Undoでは、ステージ状態をスナップショットとして復元する。



保存する状態:



\* プレイヤー座標

\* ハシゴ座標

\* 各窓の状態

\* 猫の状態

\* 手数

\* クリア済みフラグ



v0.1では、状態を毎手丸ごと保存してよい。



\### 7.2 Reset



Resetはステージ開始時の初期状態に戻す。

\## 11. スクリプト設計



\### 11.1 GameController.gd



ゲーム進行の中心。



責務:



\* 入力を受け取る

\* 移動可能か判定する

\* プレイヤー移動を実行する

\* ハシゴ移動を実行する

\* 掃除を実行する

\* UndoManagerに状態を保存する

\* クリア判定を行う



\### 11.2 GameState.gd



現在のステージ状態を保持する。



保持する情報:



\* stage\_id

\* player\_position

\* ladders

\* windows

\* cats

\* move\_count

\* is\_cleared



\### 11.3 UndoManager.gd



状態履歴を管理する。



API案:



```gdscript

func push\_state(state: GameState) -> void

func can\_undo() -> bool

func undo() -> GameState

func clear() -> void

```



\### 11.4 LevelLoader.gd



JSONを読み込んでGameStateを生成する。



API案:



```gdscript

func load\_level(path: String) -> GameState

```



\### 11.5 LocalizationManager.gd



文字列キーから現在言語のテキストを返す。



API案:



```gdscript

func set\_language(lang: String) -> void

func tr\_key(key: String) -> String

```



注意:



Godot標準の翻訳機能を使ってもよいが、v0.1では単純なJSON方式でもよい。



\### 11.6 GridTypes.gd



セル種別や定数をまとめる。



例:

\## 13. v0.1ステージ案



\### 13.1 1-01 最初の窓 / First Window



目的:



\* ハシゴを登る

\* 窓を掃除する



概要:



ハシゴは最初から汚れ窓の下にある。



プレイヤーは登って掃除するだけ。



```text

........

........

...D....

........

...H....

GGGPGGGG

```



\### 13.2 1-02 少し右へ / A Little Right



目的:



\* ハシゴを横に動かす



概要:



汚れ窓がハシゴより1マス右にある。



プレイヤーはハシゴを右へ押してから登る。



\### 13.3 1-03 ふたつの窓 / Two Windows



目的:



\* ハシゴを再利用する



概要:



汚れ窓が2つある。



1つ掃除したあと、ハシゴを移動してもう1つ掃除する。



\### 13.4 1-04 足場へ / Step Off



目的:



\* ハシゴからledgeに移る



概要:



ハシゴを登って途中の足場へ移動し、そこから窓を掃除する。



\### 13.5 1-05 猫の見る窓 / Cat’s Window



目的:



\* 猫の視線をヒントとして使う



概要:



猫がある窓を見ている。



その方向に汚れ窓がある。



プレイヤーは猫の視線を見て、掃除すべき位置に気づく。

Codexは以下の順番で実装すること。

\* 汚れ窓を配置

\### Step 8: クリア判定

\### Step 9: Undo / Reset

\### Step 10: 5ステージ化



\* 1-01から1-05まで作る

\## 15. コーディング方針



\### 15.1 GDScript



\* Godot 4.6向けのGDScriptで書く

\* できる範囲で型注釈を付ける

\* 巨大な1ファイルにしない

\* 盤面ロジックと表示ロジックをなるべく分ける



\### 15.2 物理演算を使わない



このゲームは物理アクションではない。



\* RigidBody2Dは使わない

\* CharacterBody2Dも必須ではない

\* 基本はNode2Dと座標変換で管理する

\## 17. 重要なデザイン原則



\### 17.1 1ステージ1ひらめき



基本は1ステージにつき1つの気づきを置く。



序盤は特に、複数の難しい要素を混ぜすぎない。



\### 17.2 文字より配置で伝える



パズルの答えを文章で説明しない。



画面配置で誘導する。



\### 17.3 Undo前提で気軽に試せる



プレイヤーが失敗を怖がらないようにする。



Undoは常に見える位置に置く。



\### 17.4 猫はヒントであり、感情であり、将来の障害物



v0.1では猫は動かないが、重要な存在に見えるように配置する。



猫はゲームのかわいさだけでなく、視線による誘導にも使う。



\### 17.5 パズルの見た目を明るくする



難しいパズルでも、見た目は親しみやすくする。



外壁、窓、猫、ハシゴ、掃除道具で、硬い論理パズルをやわらかく包む。
